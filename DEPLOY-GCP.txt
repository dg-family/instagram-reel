Google Cloud Run 部署（專案：ig-reel-translate）
========================================

本機無法替你完成「登入 Google」與「扣款授權」，請在你自己的電腦終端機執行下列步驟。

一、建立 GCP 專案（若尚未建立）
--------------------------------
1. 開啟 https://console.cloud.google.com/
2. 建立專案，專案 ID 請設為：ig-reel-translate
3. 連結帳單帳戶（Billing）

二、安裝 gcloud
---------------
https://cloud.google.com/sdk/docs/install

三、登入並指定專案
-----------------
  gcloud auth login
  gcloud config set project ig-reel-translate

四、一鍵部署（在專案根目錄）
---------------------------
  cd /Users/yui-liangchen/.cursor/projects/empty-window/instagram-video-cn
  chmod +x scripts/deploy-cloud-run.sh
  ./scripts/deploy-cloud-run.sh

腳本會：
  - 啟用 Cloud Run、Artifact Registry、Cloud Build
  - 建立 docker repository：ig-api（區域預設 asia-east1）
  - 用 Cloud Build 建映像並推到 Artifact Registry（:latest）
  - 部署 Cloud Run 服務名稱：ig-reel-translate-api（4Gi / 2 CPU / timeout 3600s）
  - 列印 HTTPS 網址（形如 https://ig-reel-translate-api-xxxxx-xx.a.run.app）

若要改區域或服務名稱，執行前可設環境變數：
  export GCP_REGION=us-central1
  export CLOUD_RUN_SERVICE=my-api
  ./scripts/deploy-cloud-run.sh

五、接 GitHub Pages
-------------------
1. GitHub repo → Settings → Secrets and variables → Actions
2. 設定 PUBLIC_API_URL = 上一步印出的 Cloud Run 網址（不要結尾 /）
3. Actions → Deploy GitHub Pages → Run workflow

六、僅用 Cloud Run、不用 GitHub Pages
------------------------------------
瀏覽器直接開 Cloud Run 網址根路徑即可（API 與網頁同源）。

七、在 GitHub Actions 上部署（本機不必裝 gcloud）
-----------------------------------------------
1. 在 GCP Console 建立服務帳號（例：github-deploy-ig-reel），並授予專案層級角色（擇一策略）：
   - 簡化（開發用）：「編輯者 Editor」roles/editor
   - 較精簡：至少需能送 Cloud Build、寫 Artifact Registry、部署 Cloud Run，例如：
     roles/run.admin、roles/artifactregistry.admin、roles/cloudbuild.builds.editor、
     roles/iam.serviceAccountUser、roles/storage.admin
2. 為該服務帳號建立 JSON 金鑰（Console → IAM → 服務帳號 → 金鑰 → 新增 JSON），下載檔案。
3. GitHub repo → Settings → Secrets and variables → Actions → New repository secret
   - 名稱：GCP_SA_KEY
   - 值：整份 JSON 內容（貼上即可）
4. Actions →「Deploy Cloud Run」→ Run workflow
5. 成功後日誌最後會印出 Cloud Run URL；將該網址設為 PUBLIC_API_URL，再跑「Deploy GitHub Pages」。

若 Cloud Build 在「推映像到 Artifact Registry」失敗，請在 Cloud Shell 執行（區域與 repo 名需與 workflow 一致）：

  PROJECT_NUMBER=$(gcloud projects describe ig-reel-translate --format='value(projectNumber)')
  gcloud artifacts repositories add-iam-policy-binding ig-api \
    --location=asia-east1 \
    --project=ig-reel-translate \
    --member="serviceAccount:${PROJECT_NUMBER}@cloudbuild.gserviceaccount.com" \
    --role="roles/artifactregistry.writer"
